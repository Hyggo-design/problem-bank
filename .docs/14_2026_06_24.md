## 2. Bước Ngoặt Thiết Kế (từ brainstorm)
Hình dung ban đầu ("2–3 khung tài liệu dựng sẵn trong code") **SAI** so với cách Thầy thật sự làm. Thầy mô tả workflow:
- Một **thư mục dự án LaTeX**: `main.tex` (preamble đồ sộ tự chế — docclass `book`, `ex_test[loigiai]`, watermark, lệnh `\begin{name}`/`\begin{bt}`/`\cautn`…) + thư mục con `data/` chứa **file nội dung**; `main.tex` gọi bằng `\input{data/…}`.
- **File nội dung** = khối `\begin{name}…\end{name}` (header) → dòng `%Từ đây tôi bắt đầu gõ bài tập nè` → các bài.
- 2 skeleton: **Phiếu BT** (name 3 ô) và **Đề thi** (name 6 ô).

**⟹ App phải xuất đúng FILE NỘI DUNG** (fragment để `\input`), không phải tài liệu độc lập. Đây là lý do **bỏ** `handleFinalExport` cũ.

### Quyết định chốt (Teach Me Why)
1. **Xuất file nội dung, BỎ tài liệu độc lập cũ** — khung `\documentclass{article}` cũ không khớp `main.tex` thật (docclass `book`) ⇒ vô dụng.
2. **Template = file `.tex` Thầy tự soạn, app ĐỌC từ thư mục cấu hình** — app không "đẻ" LaTeX cứng; chỉ điền + chèn bài.
3. **App tự dựng form header từ template**: đọc các dòng `{} %Nhãn` trong `\begin{name}` → form (Phiếu BT 3 ô / Đề thi 6 ô, tự thích nghi).
4. **Cấu hình thư mục template 1 lần (Cài đặt) + Save As mỗi lần xuất** — Save As cho Thầy lưu thẳng vào `data/` đúng dự án + đặt tên.
5. **`buildProblemTex` GIỮ NGUYÊN — sacred** (dùng chung Mã LaTeX + Xem đầy đủ + xuất mới); chứng minh byte-identical bằng `git diff`.
6. **Golden-file test pin định dạng file nội dung mới** (thay vai "byte-identical vs khung cũ" vì khung cũ đã bỏ).
