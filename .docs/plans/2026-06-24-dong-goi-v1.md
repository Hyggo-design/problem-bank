# Đóng gói v1.0 + Chuyển DB sang ổ D — Build Plan

**What we're building:** Dữ liệu lưu ở `D:\ProblemBank` (thay ổ C), cấu hình danh tính app, rồi đóng gói thành **installer** chạy độc lập.

**Why:** Ổ C đã đầy → data sang ổ D; và có bộ cài để dùng app như phần mềm thật (không cần `npx tauri dev`).

**Approach:** Làm phần DB-sang-ổ-D trước (lệnh Rust `ensure_dir` → `db.js` nạp DB ở ổ D, có fallback ổ C → Cài đặt hiển thị đúng), rồi sửa `tauri.conf.json`, cuối cùng `npx tauri build`. **Không đụng** schema/nội dung DB/xuất — chỉ đổi *nơi lưu* file.

**Files we'll create or change:**
- `src-tauri/src/lib.rs` — sửa: thêm `ensure_dir`, bỏ `get_db_path` (+ bỏ `use tauri::Manager`)
- `src/utils/db.js` — sửa: nạp DB từ `D:\ProblemBank` (fallback ổ C)
- `src/components/SettingsPage.jsx` — sửa: hiện đường dẫn DB từ `pb-db-path-active`
- `src-tauri/tauri.conf.json` — sửa: identifier/version/kích thước cửa sổ

> **Quy ước chung:**
> - **Frontend:** `$env:CI="false"; npm run build` → 0 warning. **Rust:** `cargo check --manifest-path src-tauri/Cargo.toml` → `Finished`. **Golden:** `$env:CI="true"; npm test` → PASS.
> - **Nhánh:** `feat-dong-goi-v1` (tách `master`).
> - **Chạy thật (Thầy):** `npx tauri dev`. **Đóng gói (em):** `npx tauri build` (lâu — build release).
> - **KHÔNG đụng** `buildProblemTex`/xuất/schema/nội dung DB. Backup DB hiện có trước khi đổi chỗ (an toàn).

### Task 1: Lệnh Rust `ensure_dir` (+ bỏ `get_db_path`)

**What you'll have when this is done:** Phía Rust có lệnh tạo thư mục (để tạo `D:\ProblemBank`); lệnh `get_db_path` cũ (trỏ ổ C, nay vô nghĩa) được gỡ.

- [ ] Bước 1: Tạo nhánh + **sao lưu DB hiện tại** (phòng hờ).
      Run: `git checkout -b feat-dong-goi-v1`
      Run: `Copy-Item "$env:APPDATA\com.tauri.dev\problem_bank.db" "$env:APPDATA\com.tauri.dev\problem_bank.before-v1.db" -Force` *(nếu không có file thì bỏ qua — chưa chạy app lần nào)*

- [ ] Bước 2: Mở `src-tauri/src/lib.rs`. Bỏ dòng `use tauri::Manager;` (sắp hết chỗ dùng) — đổi 2 dòng đầu:
      ```rust
      use std::fs;
      use tauri::Manager;
      ```
      thành:
      ```rust
      use std::fs;
      ```

- [ ] Bước 3: **Thay** hàm `get_db_path` bằng `ensure_dir`. Tìm khối:
      ```rust
      // Đường dẫn file DB (để hiển thị + làm nguồn sao lưu).
      #[tauri::command]
      fn get_db_path(app: tauri::AppHandle) -> Result<String, String> {
          let dir = app.path().app_data_dir().map_err(|e| e.to_string())?;
          Ok(dir.join("problem_bank.db").to_string_lossy().to_string())
      }
      ```
      thay bằng:
      ```rust
      // Tạo thư mục (đệ quy) nếu chưa có — dùng cho thư mục DB trên ổ D.
      #[tauri::command]
      fn ensure_dir(path: String) -> Result<(), String> {
          std::fs::create_dir_all(&path).map_err(|e| e.to_string())
      }
      ```

- [ ] Bước 4: Trong `generate_handler!`, đổi `get_db_path` → `ensure_dir`:
      ```rust
      .invoke_handler(tauri::generate_handler![
          list_content_templates,
          read_text_file,
          write_text_file,
          ensure_dir,
          copy_file,
          open_path
      ])
      ```

- [ ] Bước 5: Kiểm Rust + lưu.
      Run: `cargo check --manifest-path "src-tauri/Cargo.toml"`
      You should see: `Finished`, **không** dòng `error` và **không** cảnh báo `unused import: tauri::Manager` (nếu còn cảnh báo này → Bước 2 chưa bỏ đúng).
      Run: `git add . ; git commit -m "feat(tauri): ensure_dir (tao thu muc DB), bo get_db_path" -m "Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>"`

### Task 2: `db.js` nạp DB từ ổ D (có fallback ổ C)

**What you'll have when this is done:** App tạo & dùng DB ở `D:\ProblemBank\problem_bank.db`; nếu máy không có ổ D thì tự quay về ổ C (không chết).

- [ ] Bước 1: Mở `src/utils/db.js`. Thêm import ở đầu (dưới dòng import Database):
      ```js
      import Database from '@tauri-apps/plugin-sql';
      import { invoke } from '@tauri-apps/api/core';
      ```

- [ ] Bước 2: Thay dòng `const db = await Database.load('sqlite:problem_bank.db');` bằng:
      ```js
        // Nạp DB từ ổ D (Thầy yêu cầu — ổ C đầy). Tạo thư mục trước; lỗi (không có ổ D) -> quay về thư mục app trên C.
        const folder = localStorage.getItem('pb-db-folder') || 'D:\\ProblemBank';
        let db;
        try {
          await invoke('ensure_dir', { path: folder });
          db = await Database.load(`sqlite:${folder}\\problem_bank.db`);
          localStorage.setItem('pb-db-path-active', `${folder}\\problem_bank.db`);
        } catch (e) {
          console.warn('Không dùng được thư mục DB trên ổ D, quay về mặc định (ổ C):', e);
          db = await Database.load('sqlite:problem_bank.db');
          localStorage.removeItem('pb-db-path-active');
        }
      ```
      (Phần còn lại của `getDb` — tạo bảng, migration, seed — giữ nguyên, vẫn dùng biến `db`.)

- [ ] Bước 3: Kiểm biên dịch + golden.
      Run: `$env:CI="false"; npm run build` → `Compiled successfully.` 0 warning.
      Run: `$env:CI="true"; npm test` → 2 test PASS (định dạng xuất không đổi).

- [ ] Bước 4: Lưu tiến độ.
      Run: `git add . ; git commit -m "feat(db): nap DB tu o D (D:\\ProblemBank) co fallback o C" -m "Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>"`

### Task 5: Kiểm tổng + đóng gói + nghiệm thu

**What you'll have when this is done:** Một bộ cài (`.exe`/`.msi`) trong `src-tauri/target/release/bundle/`, và xác nhận DB nằm ở ổ D.

- [ ] Bước 1: **Kiểm tổng trước khi đóng gói.**
      Run: `cargo check --manifest-path "src-tauri/Cargo.toml"` → `Finished`.
      Run: `$env:CI="false"; npm run build` → 0 warning.
      Run: `$env:CI="true"; npm test` → PASS.
      Run: `git diff master -- src/utils/buildProblemTex.js` → **rỗng** (xuất không đổi).

- [ ] Bước 2: **Đóng gói** (lâu — build Rust release lần đầu có thể 10–20 phút).
      Run: `npx tauri build`
      You should see (cuối): dòng kiểu `Finished ... bundle ...` + đường dẫn các file trong `src-tauri\target\release\bundle\` (thư mục `nsis\…_x64-setup.exe` và `msi\….msi`).
      *Nếu lỗi đỏ:* dừng, chép nguyên dòng lỗi để xử lý (đừng sửa lung tung).

- [ ] Bước 3: **Nghiệm thu DB ở ổ D (Thầy — `npx tauri dev`)**:
      1. Chạy `npx tauri dev`.
      2. Mở **Cài đặt → Vị trí dữ liệu & sao lưu**: đường dẫn hiện `D:\ProblemBank\problem_bank.db`. Bấm **Mở thư mục** → Explorer mở `D:\ProblemBank`, có file `problem_bank.db`.

      4. Cho vào giỏ → **Xuất** 1 file nội dung (chọn template) → mở file thấy công thức nguyên vẹn (xuất vẫn chuẩn).

- [ ] Bước 4: **Cài thử installer (Thầy)**:
      1. Mở `src-tauri\target\release\bundle\nsis\` → chạy file `…-setup.exe` (Windows cảnh báo "unknown publisher" → *More info → Run anyway*).
