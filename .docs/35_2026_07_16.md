## Chốt thiết kế (spec `2026-07-16-ghi-tron-goi-transaction-design.md`)
- **Cơ chế**: lệnh Rust `execute_tx(dbPath, statements)` chạy cả cụm trong **1 transaction · 1 kết nối** (`BEGIN IMMEDIATE … COMMIT`; lỗi → `ROLLBACK`). Dùng **`rusqlite` bundled**.
- **KHÔNG schema / KHÔNG migration / KHÔNG đụng xuất `.tex` / KHÔNG đổi câu SQL hay thứ tự tham số** — chỉ đổi **CHỖ chạy** (JS `db.execute` → Rust transaction). Nội dung lưu ra **giống hệt từng byte**.
- Điểm chốt đúng-đắn: `execute_tx` mở **đúng file DB** nhờ `PRAGMA database_list` (không đoán path — quan trọng cho cả nhánh ổ D lẫn fallback ổ C).

## Ghi chú kỹ thuật
- **Vì sao rusqlite (không mò ruột plugin)**: rusqlite 0.32 khớp đúng `libsqlite3-sys 0.30` với sqlx → Cargo gộp một bản, không phát sinh 2 SQLite. `Connection::open(path)` nhận **đường dẫn thuần** → chắc trên Windows (path có dấu cách/backslash), tránh rắc rối connection-string kiểu URI của sqlx.
- **IPC**: `invoke('execute_tx', { dbPath })` → Rust `db_path` — đúng quy ước Tauri v2 (camelCase JS → snake_case Rust mặc định; đã tra docs xác nhận). Đây là arg **nhiều-từ đầu tiên** của dự án.
- `busy_timeout 5s` nhường khi kẹt tạm; **không** đổi journal mode (giữ mặc định, tránh ảnh hưởng auto-backup copy).
- Sao lưu DB dù **không** đổi schema. `src-tauri/Cargo.toml` lần này **có commit** (thêm `rusqlite` — thay đổi thật, không chỉ CRLF). Spec dở `2026-07-13-cai-thien-ux-ui-premium-design.md` (Thầy soạn): **KHÔNG đụng**.
