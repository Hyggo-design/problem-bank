## 5. Tính Năng Thứ Hai Cùng Phiên — Editor LaTeX Nâng Cao (CodeMirror)

Sau khi chốt Bảng Thống Kê, Thầy yêu cầu làm **Editor LaTeX nâng cao**: biến ô nhập LaTeX (đang là `<textarea>` chữ đen một màu) thành "IDE thu nhỏ" — tô màu cú pháp, tự đóng ngoặc, tự cặp `\begin\end`. Đủ vòng brainstorm → spec → plan → Claude tự build → Thầy *"ổn áp"*.

### Phát hiện quan trọng lúc brainstorm
Thầy kỳ vọng editor xịn hơn ⇒ file `.tex` xuất ra ổn định hơn về **căn lề tab / xuống dòng**. Xác minh code: `buildProblemTex.js` **đã tự căn lề tab + dựng khung** `\begin{bt}` bất kể ô nhập lộn xộn hay ngay ngắn — nên căn lề của file xuất VỐN đã ổn định, editor **không** đổi (và không được đổi) phần đó. Giá trị thật của editor: (a) ít lỗi ngoặc/môi trường ⇒ `.tex` ít lỗi biên dịch; (b) nhìn ra lỗi sớm nhờ màu. Hỏi Thầy nỗi đau chính → Thầy chọn **"khối chữ đen khó đọc"** ⇒ lấy tô màu làm trọng tâm.

### Thiết kế đã duyệt
Qua hỏi-đáp: (1) nền = **CodeMirror 6** (loại Monaco vì nặng + khó cấu hình worker trong CRA; loại bản-nhẹ Prism vì tự-đóng-ngoặc phải viết tay, kém chắc); (2) áp dụng **cả 3 ô** nhập LaTeX (Add / Edit / SmartImport); (3) **gác YAGNI**: live-preview kiểu Overleaf + autocomplete gõ-gợi-ý. Spec `.docs/specs/2026-07-03-editor-latex-nang-cao-design.md`, plan `.docs/plans/2026-07-03-editor-latex-nang-cao.md` (6 task).

### Build (Claude tự làm, Thầy chọn thay vì Antigravity)
